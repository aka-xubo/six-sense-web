# Insights 分析 Prompt 模板

## 目标

为浏览器历史页面生成结构化的内容洞察，帮助用户快速判断页面是否值得继续阅读、收藏、复访或用于后续工作。

## 输出格式

必须输出纯 JSON，不要输出 Markdown、解释文字或代码块。

```json
{
  "summary": "一句话摘要",
  "user_intent": "用户访问这个页面的可能意图",
  "key_points": ["要点1", "要点2", "要点3"],
  "value": "这个页面对用户的价值",
  "next_action": "建议用户下一步怎么处理",
  "type": "页面类型",
  "keywords": ["关键词1", "关键词2", "关键词3"]
}
```

## 字段规范

### summary

- 20 到 60 个中文字符，或 40 到 120 个英文字符
- 用陈述句说明页面的核心内容
- 不要使用“这是”“本文”“该网页”等空泛开头
- 优先体现具体主题、对象、问题或结论

### user_intent

- 描述用户访问页面时可能想完成的任务
- 例如：了解方案、排查错误、对比工具、查看项目、跟进资料、学习概念

### key_points

- 3 到 5 条
- 每条应是具体信息点，不要写泛泛评价
- 如果页面信息不足，可以根据标题、URL、正文片段谨慎推断

### value

- 说明页面对用户的实际价值
- 可以从学习、决策、复用、排障、跟进、收藏等角度判断

### next_action

- 给出一个具体、可执行的下一步
- 例如：继续阅读、收藏备用、整理笔记、尝试代码、加入黑名单、稍后复查

### type

从以下类型中选择一个：

- 文档
- 文章
- 问答
- 代码
- 视频
- 新闻
- 工具
- 社区
- 其他

### keywords

- 恰好 3 个
- 避免重复和无意义词
- 优先选择页面中的技术术语、产品名、主题词或动作词

## 特殊情况

如果页面无法访问、内容太少或需要登录，仍然输出完整 JSON，并在 `summary`、`value`、`next_action` 中说明限制。
